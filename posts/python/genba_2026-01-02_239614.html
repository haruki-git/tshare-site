はじめに：役割の全体像

今回できるようになることは、Pythonで使う主な記号の意味と基本的な使い分けを理解して、簡単なコードを書いて動作確認できるようになることです。

丸括弧()は関数呼び出しやタプル（複数値のグループ化）に使います。角括弧[]はリストとインデックス指定に使います。波括弧{}は辞書（キーと値）と集合に使います。

コンマ,は要素の区切りに使います。ドット.は属性やメソッドの呼び出しに使います。最初は用途ごとに覚えると混乱が減ります。

最小コードで動かして覚える

最小コードは、関数呼び出し、リストのインデックス参照、タプル・辞書の作成と参照を順に確認できる内容です。

# demo_symbols.py
def greet(name):
    return f"Hello, {name}"  # () と {}（f文字列の波括弧）

names = ["Alice", "Bob"]      # [] リスト
pair = ("x", 1)              # () タプル
info = {"a": 1, "b": 2}      # {} 辞書

print(greet(names[0]))       # ドットと()、インデックス[]
print(pair, type(pair))      # コンマで複数値
print(info["a"])             # 辞書参照 []

実行方法は次のとおりです。

  1. ファイル名を demo_symbols.py にして保存します。
  2. ターミナルで python demo_symbols.py を実行します。
  3. または VSCode でファイルを開き、上部の実行ボタンで実行します。

出力例は次のとおりです。

Hello, Alice
('x', 1) <class 'tuple'>
1

確認方法として、print を使って途中結果を表示します。type() で型を確認します。names[0] や info["a"] を個別に print して、値の取り出しを確認します。

使い分けと具体ルール

使い分けの基準は、要素を変更する予定があるならリスト [] を選ぶことです。順序が不変ならタプル () を候補にします。キーで値を参照するなら辞書 {} を選びます。要素が重複しない集合が必要なら set() を使います。関数呼び出しは必ず () を付けます。

コンマの罠は単一要素のタプルです。("a") は文字列ですが、("a",) は1要素タプルです。辞書リテラルと集合リテラルは {} で見た目が同じに見える点が紛らわしいです。{} にコロンがあると辞書になります。属性アクセスのドットはメソッド呼び出しとプロパティ取得を区別するので、メソッドは必ず () を付けて実行します。

よくあるミスは、1要素タプルのコンマを書き忘れて ("a") としてしまうことです。直し方は、要素の後ろにコンマを付けて ("a",) と表記することです。

("a")    # 文字列です
("a",)   # 1要素タプルです
{"a": 1} # コロンがあるので辞書です

デバッグの見方と1段ステップアップ

デバッグの見方は、まず print() で変数の中身と型を表示すると把握しやすいです。

print(f"names={names}, type={type(names)}")

次に logging モジュールを使うと、レベル別に出力を整理できます。

VSCode のデバッガーではブレークポイントを置き、変数ウィンドウで値を確認すると状況を追いやすいです。

1段ステップアップとして、関数化して例外処理を加える練習をすると保守しやすくなります。

例えば辞書参照を関数にして KeyError を捕まえると、処理が堅牢になります。

ファイル入出力が必要な場合は open() を with 文で使うと自動でクローズされます。

外部ライブラリが必要な場合は pip install で導入できます。

例（簡単な例外処理）です。

def get_val(d, k):
    try:
        return d[k]
    except KeyError:
        return None

注意（ここだけ）

リストや辞書をループで回しながら同じオブジェクトを変更すると、期待した順序や要素が変わることがあります。

  • 例えば、forで回している最中にリストの要素を削除すると、要素が一部飛ばされることがあります。

要約

  • 丸括弧 () は関数呼び出しとタプルの作成に使います。
  • 角括弧 [] はリストの作成と、インデックス参照やスライスに使います。
  • 波括弧 {} は辞書リテラルと集合リテラルに使うため、違いを見分けて使います。
  • コンマ , は要素の区切りに使い、単要素タプルでは末尾にコンマが必要です。
  • ドット . は属性やメソッドの参照に使い、メソッドは () を付けて呼びます。
